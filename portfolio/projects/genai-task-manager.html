01 Overview

Built during an AI Developer internship at Seventh Horse Agri-Tech (MATIC), this platform gives teams a way to manage tasks, approvals, and workflows through plain-language requests instead of clicking through a rigid task-tracker UI. Behind a conversational front end sits a full task management backend — assignment logic, workflow rules, approvals, and reporting.

02 The problem

Traditional task management tools require every action — assigning work, requesting an approval, checking a project's status — to go through a specific screen built for that exact action. That's fine for power users, but it adds friction for everyone else, and it means the tool can never answer a question it wasn't explicitly built to answer. The goal here was a system that could understand a natural-language request like "reassign this week's overdue tasks to whoever has capacity" and act on it directly.

03 Architecture

01
Conversational assistant layer
An LLM-driven assistant, shaped through prompt engineering, that interprets natural-language requests and maps them to concrete backend actions.
02
Task & workflow engine
Handles assignment logic, workflow automation, and approval routing, exposed to the assistant and the dashboard through a shared REST API.
03
Role-based access control
Every action — viewing, assigning, approving — is scoped to what a given role is actually permitted to do.
04
Data layer
Employee, project, and task data persisted in SQL, queried both by the dashboard and by the assistant when answering a request.

04 What it handles

  • Intelligent, natural-language-driven task assignment
  • Workflow automation and approval management
  • Project tracking with role-based visibility
  • Real-time monitoring of task and project status
  • On-demand report generation from conversational requests
